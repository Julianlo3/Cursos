Conceptos básicos

Git es un control de versiones que permite hacer copias de elementos generalmente de código, para guardar un historial. Inventor: Linus Torval

Para el manejo de git se hace necesario el control por consola. En git se usa el Git Bash

Ramas en git

Las ramas, como su nombre indica, son separaciones. Aquí se habla del concepto de flujo

Conceptos y comandos de las ramas
comando/concepto Descripción Otra información
git branch ... Con branch se crea una nueva rama.
git switch ... Cambio de rama
git merge ... Cuando se están en ramas y se quieren traer los cambios de un commit. Se traen los datos de los archivos Tener en cuenta los conflictos, dos personas no pueden editar el mismo archivo y la misma linea de código
git stash No hacer un commit, una copia en el repositorio, guarda los archivos de manera local.Esto con el fin de si se esta en una rama y se cambia a otra no se pierdan los cambios que se hayan hecho
  • git stash list - ver los cambios guardados en stash
  • git stash pop - traer los cambios que se guardaron
  • git stash drop - borrar los stash
git branch -d ... Se eliminan las ramas. Esto se hace porque una rama es algo temporal. Cuando ya se acaba la creación de la función se pasa al main y esta queda sin hacer nada.

Git-Hub

Plataforma que se usa con git pero en la nube, se usa para el trabajo en equipo. Resumen de los comandos básicos:

Subir proyecto a gitHub

Para enlazar el proyecto local y el proyecto en la nube (gitHub) se puede hacer dos cosas:

  1. Descargar gitHub desktop y hacerlo todo automatico
  2. Si ya se tiene un git creado, se debe usar el agento SSH. siguiendo la guía: Guia para HSS

Ya hecho el enlace de HSS si se hizo de forma manual, se crea el repositorio en gitHub y mediante consola y el gitbash se copia el comando que sale en el repositorio de gitHub.

Trabajo remoto

Así como git tiene comandos para el trabajo local, ahora que se tiene el repositorio en GitHub hay otras cosas que tener en cuenta.
