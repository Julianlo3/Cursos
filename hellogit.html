Curso de git-principios

Conceptos básicos

Git es un control de versiones que permite hacer copias de elementos generalmente de código, para guardar un historial. Inventor: Linus Torval

Para el manejo de git se hace necesario el control por consola. En git se usa el Git Bash

Comandos bácicos de consola
Comando Descripción Otra información
ls Lista los elementos de la carpeta actual
cd Cambio de directorio (posición) Puede ser posición absoluta (dirección completa)
Puede ser relativa
  • Posición actual ( ./ )
  • Pósición atrás ( cd ..)
  • Cambio posición ( cd directorio/)
pwd Mostrar toda la ruta del directorio actual
touch Creación de cualquier archivo: touch "ejemplo.txt"
mkdir Creación de carpetas Cuando se crea no se cambia automaticamente a la carpeta creada. Toca usar cd
Comandos básicos de git
Comando Descripción Otra información
  • git config --global user.name "..."
  • git config --global user.email "..."
 Estos dos comandos son los primeros pasos para iniciar con git. Siempre se necesita un usuario y un correo Los datos quedan guardados localmente y se pueden ver en .gitconfig que se puede ver su ruta con git config --global --list --show-origin
init El comando más importante, indica que el directorio actual será un repositorio Se crea un fichero oculto llamado .git en el directorio actual

Cuando se crea por primera vez, se tiene la rama principal master o main
git status Sirve para ver el estado actual, ver archivos si han cambiado
git add Añadir archivos para capturarlos Se puede seleccionar uno por uno de los elemtos o colocar git add . para agregar todo
git commit -m "nombre de la version" los commits son lo más fundamental de git, estos son los que capturan y guardan los elementos con un hash único Cada vez que se quiera hacer un commit se debe usar add pero se puede resumir con
git commit -a -m "..."
git checkout ... Vuelve a los valores del último commit guardado de un archivo en concreto
git reset Sirve para deshacer commits y ramas completas
alias. Un alias es super útil para recortar muchos comandos seguidos Por ejemplo. Si se quiere acortar la linea: git log --graph
Se coloca de forma global: git config --global alias.arbolFlujo "log --graph"
y solo se escribiria git arbolFlujo
.gitignore Aunque no sea un comando, este archivo es esencial para archivos residuales o que no querramos subir Ya creado el .gitignore en el repositorio se pone **/.. para poner extesiones o nombres de archivos a ignorar
git log
git diff 		Estos comandos son para ver el historial: git log sirve para ver el historial de los commits y
git diff para ver que se agregó o quitó de los archivos
